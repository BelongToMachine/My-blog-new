# Design Context

## Users

Recruiters, hiring managers, and fellow developers evaluating Jie's frontend engineering skills and design sensibility. They browse this portfolio to quickly assess technical depth, attention to detail, and creative problem-solving.

## Brand Personality

**Retro-pixel technical portfolio.** Playful precision. Bilingual (Chinese-first, English-secondary). Confident without being arrogant.

The entire site speaks in a unified pixel visual language — there is no "Normal mode" alternative. Pixel fonts, hard edges, zero border-radius, and grid textures are the baseline, not a toggle.

## Aesthetic Direction

- **Pixel-first everywhere**: `retro-pixel` font for headings, labels, navigation, buttons, and meta text. Zero border-radius across the board. Grid and scanline textures where appropriate.
- **Content areas stay clean and minimal**: Technical content (code blocks, articles, blog posts) should feel pure and uncluttered. Decorative chrome (macOS-style title bars, heavy badges, line numbers) is stripped away. The content itself is the design.
- **Cyan / teal primary family** unifies light and dark modes. Signal colors (amber, rose, green) are reserved for status, feedback, and accents.
- **Dark mode is not an afterthought**: Both themes are first-class citizens with full token coverage.

## Design Principles

1. **Unified pixel visual language — it's the portfolio's signature.** Every page, every component, every label should feel like it belongs to the same system.
2. **Content is king in content zones.** Articles, blogs, and code presentations should be scannable, readable, and free of unnecessary decoration.
3. **Motion should be purposeful, not decorative.** Use Framer Motion for entrance, exit, and hover states that guide attention. Avoid animation for its own sake.
4. **Maintain bilingual support throughout.** Chinese-first copy with English where it adds clarity.
5. **Every page should feel like a deliberate design choice, not a template.** No generic AI aesthetic. No stock component look.
6. **Accessibility is non-negotiable.** Target WCAG AA as the baseline for color contrast, keyboard navigation, semantic HTML, and focus states.

## Current Priority

**Styling Normalization** — progressively migrating the project onto a consistent styling architecture:

- Tailwind CSS as the primary styling layer
- Semantic CSS custom properties as the token source of truth
- `app/components/ui/*` as the reusable primitive layer
- CSS Modules only for complex isolated animations or editorial presentation
- Eliminate fragmented styling choices, ad-hoc token names, and one-off visual systems

## Typography

| Role | Font | Usage |
|------|------|-------|
| Pixel / headings | `var(--font-pixel)` | Nav, buttons, badges, labels, section headings, article H1-H3 |
| Body / sans-serif | `var(--font-sans)` | Paragraphs, descriptions, long-form reading |
| Mono / code | `var(--font-mono)` | Code blocks, inline code, terminal output |
| Decorative | `var(--font-flickering)` | Special accent moments only |

## Spacing & Shape

- **Border radius**: `0` everywhere (`--radius: 0rem`). No exceptions.
- **Shadows**: Elevated layers use `var(--shadow-elevated)` or `var(--shadow-overlay)`. No soft blurred shadows.
- **Transitions**: Default `duration-200 ease-out` for interactive states.
- **Buttons / pills**: `border-2`, `font-pixel`, `text-[10px]`, `uppercase`, `tracking-[0.22em]`, `font-medium`.
